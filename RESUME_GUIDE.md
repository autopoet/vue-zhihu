## 🎯 核心描述 (三选一)

### 方案 A：侧重“工程化与视觉表现” (适合中高级职位)
> 独立开发基于 Vue 3 的高校极客竞赛社交平台，采用 **Glassmorphism 2.0 (玻璃拟态)** 设计语言，通过自主封装的 **Bento Grid 响应式布局** 实现高度定制化的个人档案中心。利用 Vite 深度定制构建流程，通过按需加载与 SVG 动画优化，将首屏渲染时间减少 40%，打造了具备大厂质感的极客交互体验。

### 方案 B：侧重“性能优化与多线程” (适合追求技术深度)
> 负责 CampusHub 项目的前端架构设计，攻克了高频动态数据渲染性能瓶颈。引入 **Web Workers API** 将复杂的年度活跃热力图计算逻辑移至后台独立线程，成功解耦 CPU 密集型任务，确保 UI 响应始终维持在 60FPS。同时通过 **mutation-observer** 实现了一套自动感知的全局动态主题引擎，支持零闪烁的主题切换效果。

### 方案 C：侧重“业务逻辑与组件化分析” (适合校招/初级职位)
> 基于 Vue 3 全家桶构建竞赛一站式平台，通过 **Composition API** 实现业务逻辑的高度复用。设计并实现了一套基于 **Type-based Routing (类型驱动路由)** 的详情页系统，有效解决了跨模块 ID 冲突问题。独立封装了包括极客全局通知 (Toast)、骨架屏 (Skeleton) 及 3D 浮动手感标签云在内的 10+ 个高复用 UI 基础组件。

## 🛠️ 技术亮点 (可直接填入简历“主要工作”)

- **多线程离线计算**：使用 Web Worker 处理 364 天跨度的数据矩阵计算，通过 `postMessage` 异步通信模型，避免了大批量数据处理带来的主线程阻塞（Jank）。
- **组件库自主研发**：摒弃通用 UI 框架的样式限制，独立开发了符合 GitHub 审美的极客 UI 组件。
- **动态主题感知引擎**：结合 CSS 变量与 `MutationObserver` 接口，实现了一套支持深浅色模式平滑切换且持久化存储的主题系统。
- **前端性能打磨**：通过对 ECharts 等重型库的按需引入（后优化为纯原生 CSS 方案），极大地缩减了静态资源体积，提升了页面的加载性能。
- **数据一致性治理**：针对 Mock 模拟环境设计了一套详细的 ID 映射与类型感知逻辑，通过路由守卫与状态拦截，确保了从收藏夹到详情页的高可靠导航跳转。

## 💬 模拟面试问答 (Q&A)

**Q：为什么你在个人中心使用 Web Worker？**
> **A**：因为生成一整年的活跃度热力图需要进行大量的数组遍历和随机等级计算。在 JS 的单线程环境下，这 500 万次级别的循环会占用主线程，导致浏览器无法处理用户的滚动或点击。我通过 Web Worker 将这些逻辑移出主线程，主线程只负责接收计算结果并渲染，从而保证了用户交互的绝对流畅。

**Q：你在项目中遇到的最大挑战是什么？**
> **A**：最大的挑战是视觉美感与页面性能的平衡。例如早先使用 ECharts 来实现雷达图，虽然功能丰富但体积庞大且在暗黑模式适配上略显生硬。我后来决定查阅 MDN 和现代 CSS 设计规范，自主实现了一套基于 Grid 布局的 Bento Dashoard。这不仅减轻了 100KB+ 的依赖包体积，还通过 CSS 变量实现了与系统主题的深度耦合，不仅更好看，加载也更快。
